## Highlights

Fourth public beta. Two new polish features — themed colour palettes and album-cover quick-set — plus a stability round focused on Pixel 9 / Android 17 where the prior beta could crash mid-bootstrap. The cryptographic locking that v1.2 introduced around `libgojni` is now exhaustive: every call site that touches the Go runtime is serialized, removing the userfaultfd race on Android 17's strict GC.

## What's new

### Personalisation
- Six colour palettes in Settings → Appearance: Default (Proton purple), Forest, Sunset, Sea, Sepia, Mono. Status badges (Synced green, Pending amber, Error red) stay semantic and are not recoloured.
- The Material 3 `primaryContainer` is now derived from the active palette, so chips, switches and sliders pick up the chosen accent automatically.

### Albums
- Long-press any photo inside an album to open a quick-action menu with "Set as cover" and "Remove from album".
- "Set as album cover" item added to the photo viewer's overflow menu when the photo was opened from an album.
- Success toast plus automatic refresh of the albums list — the new cover thumbnail appears immediately on return.

### Stability — Pixel 9 / Android 17
- Global crypto lock now covers every direct `pgpCrypto` call in the codebase. Previously only the `DriveCryptoHelper`-owned paths were guarded; this round adds:
  - first-launch volume bootstrap (two PGP key generations + unlocks)
  - album creation and add-photos-to-album bulk decrypt loops
  - the root-link public key fetch on every upload
  - share key derivation
  - photo entity decryption inside the sync pipeline
- The lock is re-entrant so existing `DriveCryptoHelper` callers don't deadlock.

### Reliability
- Viewer race fix: late full-resolution loads no longer land on the wrong page after a fast pager swipe.
- Editor save paths now rethrow `CancellationException` correctly — backing out of an in-progress save no longer surfaces a false "Save failed" toast.
- Removed five `remember { ...; Unit }` Compose anti-patterns in the photo and video editors (these were already working but stopped firing on certain key-change paths under future Compose runtimes).
- Hardened atomic operations on the fallback-refresh cooldown gate (`AtomicLong` + CAS replaces a bare `@Volatile var`).

### Developer & community
- GitHub issue template added — a single form that handles crashes, bugs and feature requests, with built-in fields for device, Android version, OS flavour, and a logcat-capture how-to that doesn't require any third-party tools.

### Localization
- All new strings translated to German, Spanish, French, Hungarian and Italian.

## Known limitations

- Album invitation by email still returns "outdated app" from the Drive backend. The multi-step invite UI (email chips, permissions, optional message) ships, but the underlying server reply hasn't changed. Public link sharing works.
- Unified trash that mirrors device-wide deletions (like Samsung Gallery) is still pending.
- HEIC and RAW editing not yet supported.
- Cloud photo GPS is not yet stored in the encrypted xAttr, so a future map-style "Places" view will only see local + Synced photos until that schema lands.

## Install

Grab `app-universal-release.apk` from the assets below for the simplest install. For a smaller download, pick the ABI matching your device (`arm64-v8a` for most modern phones, `armeabi-v7a` for older / 32-bit, `x86_64` for emulators, `x86` for legacy x86).

Source on GitHub — GPL-3.0, no obfuscation (`-dontobfuscate`), so the released APK can be byte-compared against a fresh build from this source.
